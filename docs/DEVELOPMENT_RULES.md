# DEVELOPMENT_RULES

## Regras de Desenvolvimento

1. Nunca implementar algo fora do PRD
2. Sempre explicar a abordagem antes de gerar código
3. Criar código incremental
4. Um arquivo por vez
5. Preferir soluções simples
6. Não introduzir dependências sem justificativa
7. Não gerar código morto
8. Não refatorar sem solicitação explícita
